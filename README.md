# apbd-cw1-git-s30455

1. A fast-forward happens when the branch you're merging into has not diverged from the branch you're merging in. A merge commit is created when the two branches have diverged: each has commits the other doesn't.
2. Merge takes the two branch tips and combines them with a merge commit, the existing commits are untouched, and the history records that two lines of work happened in parallel and were joined. Rebase takes your branch's commits and replays them one by one on top of another branch.
3. A merge conflict occurred because the same variable was named differently in the two branches. I resolved the conflict manually by keeping the version from the main branch.
